## Deployment & Integrations

Complyr is a multi-system application. A good documentation set should make those boundaries obvious.

## Core Integrations

### Zama

Used for:

- client-side encrypted input creation
- ciphertext handle generation
- proof generation
- user-authorized decryption

Primary implementation file:

- `apps/web/src/lib/fhe-audit.ts`

### Privy

Used for:

- authentication
- wallet connection
- embedded wallet flows

### Pimlico

Used for:

- ERC-4337 bundling
- paymaster-sponsored transactions

### Envio

Used for:

- indexed activity feeds
- wallet dashboard read models

### PostgreSQL

Used for:

- contacts
- contact addresses
- app-owned relational data

## Recommended Documentation Convention

For each integration, the docs should answer the same four questions:

1. Why is this dependency in the architecture?
2. What exact feature depends on it?
3. What fails if it is unavailable?
4. Which files contain the implementation?

That structure is widely used in strong protocol and infrastructure documentation because it helps reviewers navigate quickly.
