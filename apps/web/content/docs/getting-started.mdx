## Getting Started

### Prerequisites

- Node.js 18+
- pnpm
- Foundry (for contract development)

### Clone and install

```bash
git clone https://github.com/Stoneybro/complyr
cd complyr
pnpm install
```

### Run the web application

```bash
pnpm dev
```

The application runs at `http://localhost:3000`.

### Environment variables

Create `apps/web/.env.local`:

```bash
# Privy authentication
NEXT_PUBLIC_PRIVY_APP_ID=your_privy_app_id

# Neon PostgreSQL (for contacts)
COMPLYR_DATABASE_URL=postgresql://...

# Envio GraphQL API (optional override)
NEXT_PUBLIC_ENVIO_API_URL=https://indexer.dev.hyperindex.xyz/63b8cce/v1/graphql

# Pimlico
NEXT_PUBLIC_PIMLICO_API_KEY=...
NEXT_PUBLIC_PIMPLICO_SPONSOR_ID=...
```

### Build contracts

```bash
# Base Sepolia contracts
pnpm forge:build

# Run tests
pnpm forge:test
```

### Run the indexer locally (optional)

If you want the app to read from your own local indexer instead of hosted Envio:

```bash
cd packages/indexer
pnpm codegen
pnpm dev
```

Then point the web app to `http://localhost:8080/v1/graphql` via `NEXT_PUBLIC_ENVIO_API_URL`.

---

### Account abstraction setup

Complyr uses Pimlico on Base Sepolia for ERC-4337 bundling and sponsorship. Bundler and paymaster configuration lives in `apps/web/src/lib/bundler.ts` and `apps/web/src/lib/smartAccountClient.ts`.

Recurring payment execution is exposed through Chainlink Automation-compatible methods (`checkUpkeep` and `performUpkeep`) in `IntentRegistry`.
